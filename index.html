Jak používat testovací reporty

Od spuštění testů až po přehledné zobrazení výsledků na GitHub Pages.

Spuštění testů

V projektu s testy spusť příkaz npx cypress run. Během běhu se každý výsledek ukládá do souboru live-log.json.

Po dokončení testování se tento JSON automaticky převede na přehledný CSV report, který obsahuje kompletní přehled všech testů.

live-log.json – průběžný výstup testů
YYYY_MM_DD_HH_mm.csv – finální report po doběhnutí

Nahrání na GitHub

Po vygenerování CSV musíš soubor ručně nahrát do projektu s výsledky:

reports/reports/<modul>/<typ>/

Dále spusť skript, který aktualizuje přehled reportů:

node generate-reports1-json.js

Jakmile se vytvoří reports.json, commitni změny do repozitáře:

git commit -am "přidán nový report"
git push

Výsledky se pak automaticky zobrazí na GitHub Pages podle URL report-readeru.

Na co si dát pozor

Názvy složek musí být malými písmeny, bez mezer a bez diakritiky.
✅ Správně: folder-management    ❌ Špatně: Správa složek

Každý test musí být ve funkci describe() a obsahovat tag: @high, @middle nebo @low.

CSV hlavička musí být přesně:
describe,test,status,
duration,timestamp,timestampLocal

reports.json se negeneruje ručně!
Použij node generate-reports1-json.js, jinak se stránky rozbijí.
